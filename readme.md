# shadow.ai

**Your AI co-worker that handles everyday tasks via Slack.**

Tag the bot in any Slack thread and it runs Claude Code on your local machine ‚Äî with access to your files, git repos, MCP tools, and more.

Created by Moksh Khajanchi

---

## Prerequisites

- **Python 3.11+** ‚Äî `python3 --version`
- **Node.js 18+** ‚Äî `node --version` (needed for Claude Code CLI)
- **Claude Code CLI** ‚Äî `npm install -g @anthropic-ai/claude-code`
- **A Slack workspace** where you can create apps

---

## Setup

```bash
git clone https://github.com/mokshkhajanchi/shadow-ai.git && cd shadow-ai
git checkout v1.0.0          # Use the latest stable release
python3.11 -m venv .venv
source .venv/bin/activate
pip install -e .
shadow-ai doctor             # Check prerequisites
shadow-ai init               # 3-step setup wizard (creates .env)
shadow-ai                    # Start the bot
```

The wizard walks you through creating a Slack app, getting tokens, and setting access control. Takes ~3 minutes.

## Updates & Releases

The `main` branch is for active development. Always install from a **tagged release** (e.g. `v1.0.0`).

**Check available versions:**
```bash
git fetch --tags && git tag
```

**Update to a new version:**
```bash
git fetch --tags
git checkout v<new-version>
pip install -e .
shadow-ai doctor   # Verify everything still works
```

Your `.env`, `knowledge/`, and local data are untouched during updates.

## Usage

Mention the bot in any Slack channel or DM:

| Command | What it does |
|---|---|
| `@bot <message>` | Ask anything ‚Äî full Claude Code access |
| `opus: <message>` | Use Claude Opus (most capable) |
| `haiku: <message>` | Use Claude Haiku (fastest, cheapest) |
| `think: <message>` | Extended reasoning mode |
| `summarize` | Recap the thread |
| `learn` / `take note` | Save curated notes from this conversation |
| `review <PR-URL>` | Review an Azure DevOps PR |
| `status` | Active sessions, cost, feedback stats |
| `kill all` | Stop all sessions |

## Configuration

All config is in `.env` (generated by `shadow-ai init`). Key variables:

| Variable | Default | Description |
|---|---|---|
| `BOT_USERNAME` | *(required)* | Your bot name ‚Üí `username.shadow.ai` |
| `SLACK_BOT_TOKEN` | *(required)* | Bot token (xoxb-...) |
| `SLACK_APP_TOKEN` | *(required)* | App token (xapp-...) |
| `ALLOWED_USER_IDS` | *(required)* | Comma-separated Slack user IDs |
| `CLAUDE_WORK_DIR` | `~/Projects` | Working directory for Claude |
| `DAILY_BUDGET_USD` | `500` | Daily cost limit |
| `SYSTEM_PROMPT_FILE` | `knowledge/system_prompt.example.md` | Custom system prompt |

See `.env.example` for the full list.

## MCP Servers (Recommended)

MCP servers are what make the bot truly powerful. The bot **auto-discovers** any MCP server configured in `~/.claude/settings.json` ‚Äî no extra setup needed.

Connect your tools to give the bot direct access:

| Server | What it unlocks |
|---|---|
| **Jira** | Create/update tickets, search issues, manage sprints |
| **Azure DevOps** | Review PRs, manage work items, search repos |
| **Sentry** | Investigate errors, list issues, analyze traces |
| **Grafana** | Query dashboards, check alerts, run Prometheus queries |
| **GitHub** | Manage repos, PRs, issues |
| **Slack** | Send messages, search channels, create threads |

Add a server to `~/.claude/settings.json`:

```json
{
  "mcpServers": {
    "jira": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@anthropic-ai/claude-code-jira-mcp", "--token", "YOUR_TOKEN"]
    }
  }
}
```

Restart the bot ‚Äî `shadow-ai doctor` will show the server count. The more tools you connect, the more useful the bot becomes.

## Agents

The bot ships with specialized agents that Claude uses automatically:

| Agent | When it's used |
|---|---|
| **code-reviewer** | PR reviews ‚Äî correctness, security, performance |
| **debugger** | Error tracing ‚Äî systematic root cause analysis |
| **note-taker** | Knowledge extraction ‚Äî saves structured notes |

Add your own agents by dropping `.md` files in `knowledge/agents/`. Format: YAML frontmatter (name, description, tools, model) + markdown prompt body.

## Knowledge Base

The bot learns and remembers across conversations:

- **Auto-save** ‚Äî every conversation is saved to `knowledge/conversations/` automatically
- **Notes** ‚Äî say `learn`, `save this`, `remember this`, `take note` (or similar) in any thread to save curated notes to `knowledge/notes/`
- **Proactive** ‚Äî saved notes are injected into every new session so the bot remembers context without being asked

Notes are markdown files ‚Äî human-readable and editable. The bot gets smarter the more you use it.

## Security Note

**Only add your own Slack user ID to `ALLOWED_USER_IDS`.** The bot runs on your local machine with full filesystem access ‚Äî anyone allowed to use it can read, modify, or delete files, run shell commands, and access any MCP tool you've connected. Treat bot access like SSH access to your machine.

## Troubleshooting

Run `shadow-ai doctor` to diagnose issues.

| Problem | Solution |
|---|---|
| `claude: command not found` | `npm install -g @anthropic-ai/claude-code` |
| Python 3.10 or lower | Install Python 3.11+ via pyenv or brew |
| Bot doesn't respond | Check `ALLOWED_USER_IDS` includes your Slack user ID |
| Socket mode error | Regenerate App-Level Token with `connections:write` scope |
