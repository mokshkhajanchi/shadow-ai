# Brainstorm

Before implementing anything, work through these steps:

## 1. Clarify the Goal
- What exactly does the user want?
- What problem does this solve?
- What does success look like?

## 2. Explore Constraints
- What existing code/patterns must this work with?
- Any performance, security, or compatibility requirements?
- What's the simplest solution that works?

## 3. Consider Approaches
- List 2-3 approaches with trade-offs
- For each: complexity, maintainability, performance
- Recommend one with clear reasoning

## 4. Identify Edge Cases
- What inputs could break this?
- What happens on failure?
- Concurrent access issues?

## 5. Output
Present a brief plan:
- **Approach**: What you'll do and why
- **Files**: What you'll modify/create
- **Edge cases**: How you'll handle them
- **Verification**: How to test it works

Do NOT start coding until the plan is clear.
